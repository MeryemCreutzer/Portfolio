Content Architecture Solution
Project (CASP)

Overview

The Content Architecture Solution Project (CASP) was designed to streamline access to educational content for teachers at RobotsMali. The primary goal was to organize instructional materials within a structured, user-friendly system, leveraging tools such as Google Drive, Google Sites, Canva, and Genially. The project introduced a modular and hierarchical approach to document organization, enabling seamless navigation and interaction.

📌 Key Features:

  • Intuitive navigation system for educational resources.
  • Multi-level content structure (Sections, Classes, Subclasses).
  • Interactive guides for engaging with learning materials.
  • Search and filtering capabilities for quick content retrieval.

Context & Background

RobotsMali is a STEM and robotics education program focused on empowering students and educators through innovative learning. Over time, the organization accumulated a large volume of lesson plans and instructional guides. These materials were dispersed across various locations, making it time-consuming and frustrating for teachers to locate what they needed. The lack of a central, structured system led to inconsistent access, duplicate efforts, and confusion in content usage. Teachers struggled to navigate and retrieve the appropriate resources during class preparation or delivery. There was a clear need for a centralized, well-structured content architecture that could support scalability and ease of use.

ASP (Content Architecture Solution Project) was developed to address these issues by providing a hierarchical, organized, and intuitive content management framework.

My Role

As the Information Architect & Project Manager, I was responsible for:

  • Leading the technical communication process, liaising between the project sponsor and development team.
  • Designing the system architecture, ensuring a clear and accessible structure.
  • Creating documentation (functional specifications, user guides).
  • Developing the navigation system, implementing a structured, easy-to-use framework for teachers.
  • Testing and refining the interface, incorporating UX principles to enhance usability.

Challenges & Solutions

Process & Methodology

User research & Requirement analysis

  • We began by analyzing how teachers accessed and used existing resources. This revealed workflow inefficiencies, such as scattered content, unclear file structures, and repetitive manual steps. These findings helped define key requirements around centralization, structure, and ease of access.

System architecture design

  • To address these needs, we designed a modular hierarchy (Sections → Classes → Subclasses) using Google Drive and Google Sites. This structure mirrored how educational content is delivered and made it easier for teachers to locate and browse materials intuitively. The system was built to be consistent across all categories.

Prototyping & Testing

  • We began with paper mockups to sketch the structure and flow of the site. Tools like XMind were used to map out the content hierarchy, while Adobe XD supported more detailed wireframing. Genially was used to create interactive diagrams embedded in the Google Site, and Canva was used to design visual learning modules. Feedback gathered during testing helped refine navigation patterns and visual structure.

Deployment & Documentation

  • Once the system was finalized, we created a dedicated help center to support users during onboarding. This included visual guides and written documentation tailored to different teachers and admin users of the solution. We also delivered detailed functional and structural specifications to the project stakeholders. These documents not only formalized the implementation but also laid the groundwork for future maintenance and scalability.

Final Outcome

The CASP solution successfully transformed content accessibility for RobotsMali teachers, offering:

  • Centralized educational content with structured navigation.
  • Interactive learning materials that improved engagement.
  • Search & filtering functionalities for quick content retrieval.
  • Comprehensive user guides to facilitate adoption.
